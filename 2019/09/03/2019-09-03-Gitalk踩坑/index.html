2019-09-03-Gitalk踩坑

发表于 2019-09-03 | 分类于 web前端 ， hexo | | 阅读次数:

本Hexo博客因为是静态博客，所以没有后台，评论系统依赖于github。目前github上有两种比较流行的博客评论系统，gitment和gitalk，但是gitment已经不再进行维护，且用户评论不进行身份验证，渐渐地淡出了人们的视线。我花了大约四个小时，将原本NexT博客框架自带的gitment修改为了gitalk。这里分享一下我的踩坑历史。

配置gitalk

在github上注册应用，链接：https://github.com/settings/applications/new

如果你在搭建gitalk之后出现了初始化登录，你登陆了又跳回首页，然后在进入文章还是要你初始化，那问题很可能就出在这里！
如果你购买了域名，然后将域名解析到用户名.github.io, Authorization Callback URL填的一定是你购买的那个域名，协议是http就是http，不能是https,有www的话就是有www, 不能省去，要保证一模一样；要是你不确定域名具体是什么，先随便给一个，出了错看控制台，根据报错的请求来修改。

验证结果

hexo clean & hexo g & hexo s 素质三连之后，输入localhost:4000查看效果，点击一篇文章，如果其最下方出现需要你初始化的消息，然后点击之后，进入首页，在进入文章出现评论区，则gitalk配置成功。

小结

这次为个人博客设置评论区，我花了前后四个多小时的时间，实际上的有效操作其实加起来也就是十分钟左右，其他时间就是在一次又一次地踩坑，通过这件事，我再一次提醒自己:1.不要带着那种一生悬命的想法取追求某个无足轻重的东西，已经不再是学生的你时间上耗不起。2. 如果是不可抗力造成的困难，没的说，就是干，但是自己不要给自己挖坑。
