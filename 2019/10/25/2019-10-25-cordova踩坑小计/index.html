cordova学习笔记

混合APP是相对于原生APP和webapp而言的。原生APP使用java、swift等语言开发，调用设备底层接口方便，但是兼容性不理想，各种平台上要进行针对性的开发；webapp基于响应式web开发，可以在浏览器上运行，更易于兼容各种平台，但是对于硬件的调用不方便；混合APP(hybridApp)使用Web前端开发界面，同构添加一个Android、IOS或者其他系统的外壳间接实现对不同平台的兼容和对硬件系统的调用，是前端的一部分。

安装

  1. 安装Android SDK（随Android studio一同安装，开始时未检测到电脑中有SDK，会报错，忽略，之后会自动下载SDK）

  2. 配置环境变量：默认为C:\Users\Administrator\AppData\Local\Android\Sdk，在系统变量中将以上变量设置为%ANDROID_HOME%，在Path中添加%ANDROID_HOME%、%ANDROID_HOME%\tools、%ANDROID_HOME%\platform-tools，在控制台输入adb，如果有反应则环境变量配置生效，有些情况下需要重启计算机环境变量才能生效。

  3. 新建cordova项目，create cordova 项目名，项目文件夹所在目录最好不要有中文等非ASCII码，否则编译可能不通过

  4. 执行后，进入项目目录，hooks、platforms、plugins都是与原生APP有关的目录，webapp内容放在www中

  5. 添加平台，cordova platform add 平台名（如 Android），执行后，platforms文件夹下会出现 Android文件夹

  6. 运行之前，先进行编译， cordova build，第一次编译可能要下载一些文件，所以需要联网。如果报错Failed to install the following Android SDK packages as some licences have not been accepted，cd到SDK下的\tools\bin目录下，运行sdkmanager –licenses指令，全部同意

  7. 安装安卓模拟器，https://www.runoob.com/w3cnote/android-tutorial-genymotion-install.html。

  8. 如果模拟器安装或者破解不成功（其实不一定要是genymotion，在360软件管家里随便一键安装一个就可以），可以使用本机调试。用数据线连接手机，启动开发者模式，打开USB调试接口；如果默认连接方式是只充电的话，要手动改成网络共享；使用360手机助手帮助我们解决很多连接上的问题，如果360手机助手能连接上，手机基本上也就能调试了。

  9. 如果出现waiting for debugger force close的提示，我的解决办法是关闭开发者模式中的等待调试程序。

  10. 执行指令 cordova run Android 在安卓平台进行安装和运行。

  11. 手机出现默认应用界面，终于实现了0到1的突破！

  12. chrome://inspect/#devices可以看到运行出来的web效果

使用

  1. cordova 对于设备 API的封装，都是以插件和事件的方式体现的

  2. cordova 中对于事件的监听方式，都类似

  3. deviceready：设备API准备就绪时执行，基于设备API的底层事件或方法，都要在deviceready事件完成后执行

  4. pause: 当应用进入后台（挂起）时执行，参数同上

  5. resume: 当应用回到最前面时，调用执行，参数同上

  6. cordova应用启动流程

    1. 原生代码启动
    2. splashscreen展示（进入APP的展示页，比如有几秒的广告之类）
    3. load HTML
    4. 顺序执行 JavaScript=> cordova.js也执行
    5. $(document).ready()执行，cordova中的事件不一定可用
    6. 注册deviceready事件

  7. cordova电池插件的使用

    1. 安装电池插件

    2. 该插件可以用来监视设备电池的变化，全局添加了三个事件：

      1. batterystatus： 电池状态发生改变，至少是1%的电量变化，或者，充电状态改变才会触发此事件
      2. batterycritical： 表示电量进入临界值(快关机了)
      3. batterylow 电池电量比较低，触发

    3. 插件的事件监听以及插件方法的调用，一定要在deviceready准备完成后调用

    4. 所有的事件都会返回一个status对象，这个对象有2个属性，

      1. status.level 表示设备的电量 0-100 一个Number类型
      2. status.isPlugged 当前设备是否正在充电，返回一个boolean, true: 在充电，false：不在充电

  8. cordova照相机

    1. cordova 中提供了，可以调用摄像头设备的API，需要安装对应的插件

    2. 安装

    3. 这个插件提供了一个全局对象，navigator.camera，提供了一系列API，这些API可以帮助我们拍照，或者从相册中读取图片
      注意： 需要在devicerea 完成后，才能获取navigator.camera 对象

    4. navigator.camera中提供了方法getPicture(成功回调函数，失败回调函数，配置参数)，可以用于拍照或者从相册中读取图片

    5. 需求

      1. 点击按钮，拍摄照片，显示在页面中
      2. 点击按钮，从相册中选择图片，显示在页面中

    6. option参数

      1. quality: 表示图片质量，默认50,最大100
      2. destinationType: 表示返回的图片格式到底是路径还是base64格式字符串
        默认值：FILE_URI 返回文件路径
      3. sourceType：设置使用摄像头，还是从相册读取
        默认值：CAMERA 使用摄像头
      4. cameraDirection: 摄像头方向：FRONT或0前置，BACK或1后置
      5. targetWidth、targetHeight 保持原有比例，进行压缩显示，缩放以短的为准
