版本管理工具初探

版本管理工具简介

版本管理工具的作用：备份、记录历史（会把每一次提交都做一个备份）、共享。

  • 集中式：需要一个中心，必须联网到中心服务器才能做
  • 分布式：每一个拥有版本库的人都可以在不联网的情况下快速完成文件的提交、删除等操作，比集中式效率高
    git和GitHub不是一个东西，GitHub是一个程序员社区网站，在这个网站上程序员你可以托管他的软件库，只是软件托管这个事是基于git完成的。

到github官网下载GitHub客户端，安装。

新建一个仓库

仓库（repository），可以理解为一个目录，这个目录里的文件都可以被管理、追踪。

  • 初始化版本库

  • 将文件添加到版本库

  • 查看仓库状态

使用过程

  1. 打开GitBash，cd到想要的目录下。
  2. 输入git init 命令，初始化版本库，出现 Initialized empty Git repository in XXX 则成功. 出现一个隐藏目录.git/
  3. 在该目录中创建一个文件text.txt,添加一些内容，输入命令 git status，发现test.txt标红，说明该文件没有添加到git中
  4. 输入命令：git add text.txt，将文件添加到版本库，再输入命令 git status，发现test.txt 呈绿色，说明该文件已经添加到git中
  5. 提交，输入命令git commit -m "repo2 first commit" 将git 提交，提交说明是“repo2 first commit”
  6. 输入命令git status，发现当前处于master分支（branch）中，出现“nothing to commit, working tree clean”，说明我们所有本地的修改都已经提交到当前分支

Git 工作流

  1. 提交一个需求过来，然后我们进行开发
  2. 添加一个文件作为我们的解决方案（假装）base_demo.txt
  3. git status ,发现base_demo.txt标红，说明这个文件在工作区，但还没有在本地仓库进行跟踪
  4. git add base_demo.txt ，将文件添加到暂存区中
  5. git status，发现base_demo.txt添加进去了。
  6. commit，将暂存区的文件添加到我们最终的本地仓库中 git commit -m “bash first commit”

现在有另外一个场景，产品经理临时变更需求，我们修改bash_demo.txt文件中的内容来假装修改了项目

  1. git status，发现base_demo.txt 标红，说明修改后还没有提交到暂存区中，那么这个时候我们文件的修改就没有记录
  2. git add base_demo.txt,将文件提交到暂存区
  3. 后来，产品经理说还是原版的好，git reset HEAD bash_demo.txt,将修改回归到工作区，昨天新提交的内容应该已经作废
  4. git checkout – base_demo.txt 将工作区文件弄干净，再打开bash_demo.txt，发现文件回到了修改之前的状态

我们开始了第二天的工作，编写代码后，添加：git add bash_demo.txt, 提交：git commit – “second commit”，然后产品经理说第二天的不用做，需要我们回滚到第一天的状态，这时候的做法。

  1. 输入git log ，我们根据每次提交的说明找到自己想要的回滚版本号(一串hash散列值)
  2. git reset –hard 81fde81bb5647f2473f79c79b495ce67aca7e137(hash散列值)，回滚到当前版本，hard意思是工作区和仓库都回滚

现在需求又变了，我们什么都不需要写，只要把我们本地仓库清理干净，其他什么都不用做了，

  1. git rm base_demo.txt 就将我们本地的base_demo.txt文件删除了


index: 暂存区
HEAD：当前仓库指针
Object：实际存储的地方，暂存区和仓库都是通过指针指向这里面的数据

  • add:将工作区里的修改提交到暂存区中，同时暂存区里的目录树会被修改然后提交到Object中，修改的文件Id会被暂存区通过指针引用。
  • commit: 将master中的指针指向暂存区中的目录或者文件
  • reset HEAD: 暂存区中的目录会被重写，本地实际上也已经有所更新了。实际上试讲最终的一次提交映射到暂存区里面，但是工作区里的文件时不做修改的。
  • checkout: 工作区里修改了一些东西，但是没有做任何提交，使用这个命令之后，所有本地仓库实际上就变成干净的。
  • rm –cached:会直接从暂存区中删除文件，就是所暂存区里没有文件了，但是工作区里还有。要想删除工作区里的文件我们需要删除之后再进行commit。
  • checkout HEAD: 用HEAD指向的分支的文件替换暂存区和工作区。（这个操作比较危险，一般不会用）
